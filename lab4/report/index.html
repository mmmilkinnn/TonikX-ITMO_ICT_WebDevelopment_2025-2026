    Лабораторная работа №4

    Описание проекта

    В рамках лабораторной работы была разработана информационная система управления отелем.
    Система предназначена для автоматизации основных процессов гостиницы: работы с клиентами, номерами, сотрудниками, бронированиями и расписанием уборок.

    Проект реализован в виде клиент-серверного веб-приложения с разделением логики на frontend и backend. Особое внимание было уделено структуре проекта, навигации и взаимодействию с API.

    Стек технологий

    Frontend: - Vue.js 3 (Composition API) - Pinia (управление состоянием) - Vuetify (UI-компоненты и стилизация) - Axios (HTTP-запросы)

    Backend: - Django REST Framework

    База данных: - PostgreSQL

    Прочее: - JWT-аутентификация - Material Design Icons

    Архитектура приложения

    Frontend-приложение построено по компонентному принципу.
    Каждая логическая сущность системы вынесена в отдельный view-компонент, что упрощает поддержку и масштабирование проекта.

    Для навигации используется Vue Router. Доступ к страницам ограничен в зависимости от наличия токена авторизации.

    Глобальное состояние (токен, данные пользователя) хранится в Pinia.

    Основные компоненты и функциональность

    1. Бронирование номеров

    Функциональность: - создание, редактирование и удаление бронирований; - привязка клиента и номера к бронированию; - указание дат заезда и выезда; - получение данных с backend API.

    Реализация: - отдельная страница бронирований; - модальные окна для добавления и редактирования; - обновление данных без перезагрузки страницы.

    2. Управление клиентами

    Функциональность: - добавление, редактирование и удаление клиентов; - отображение списка клиентов; - поиск и фильтрация данных.

    Интерфейс: - таблица клиентов; - формы ввода данных в модальных окнах; - взаимодействие с API через Axios.

    3. Управление сотрудниками

    Функциональность: - хранение информации о сотрудниках; - редактирование данных; - удаление записей.

    Реализация: - отдельный view для сотрудников; - работа с REST API; - динамическое обновление списка.

    4. Расписание уборок

    Функциональность: - создание и редактирование расписаний уборок; - назначение сотрудников на конкретные номера; - фильтрация по дате.

    Интерфейс: - таблица расписания; - формы управления данными; - синхронизация с сервером.

    Навигация

    В системе реализована боковая навигация со следующими разделами:

    • Главная страница
    • Клиенты
    • Сотрудники
    • Комнаты
    • Бронирования
    • Расписание

    Навигация доступна только после успешной авторизации пользователя.

    Особенности реализации

    • Используется централизованное хранилище состояния (Pinia);
    • Реализована работа с токенами авторизации;
    • Все данные загружаются асинхронно через API;
    • Интерфейс построен на компонентах Vuetify;
    • Проект имеет четкую структуру и разделение ответственности.

    Вывод

    В ходе выполнения лабораторной работы была спроектирована и реализована структура информационной системы управления отелем.
    Проект демонстрирует применение современных web-технологий, работу с API, компонентный подход и основы архитектуры frontend-приложений.
